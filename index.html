Pre-Execution Product Validation
Catch Errors Before They Propagate
Identifier extraction — numeric IDs, prefixed IDs, part numbers detected and validated before main reasoning
Replacement detection — deprecated products flagged with their successor before the agent answers
Inventory pre-load — active product data injected as system context, eliminating redundant searches
Confidence tiers — HIGH/MEDIUM/LOW scoring via async validation callback guides LLM certainty
🧠
Context Management Middleware
Prevent Context Degradation
Tool output trimmer — caps individual RAG results to prevent a single chunk from flooding the context window
Summarization middleware — older messages auto-summarized past token threshold; latest context preserved verbatim
Tool artifact stripping — stale tool messages removed from history to reduce noise and recency bias
Prompt reminder injection — grounding reminder auto-injected when context gets long, removed after processing
🛡️
Guardrails & Prompt Architecture
Safety + Structured Reasoning
Score-based input guardrail — LLM scores input 0–1 with mandatory rationale; requests below threshold are blocked
Output safety check — categorical verdict (SAFE / SAFE_REFUSAL / UNSAFE) on every agent response
Reasoning transparency — agents must narrate intent before every tool call; no silent, ungrounded actions
Decomposition-first prompts — complex queries split into research tasks; all tasks completed before composing a reply
📋
Codified Expert Workflow Patterns
Domain Knowledge as Agent Behavior
Named workflow patterns — e.g. configure-product, find-substitution, lifecycle-check baked into agent prompts as step-by-step instructions
Pattern invoked upfront — agent selects the right workflow pattern at the start of a turn, then executes it sequentially
Qualifier preservation — family/generation qualifiers (G2, V2, Plus) carried through every step to prevent wrong-product drift
Not improvised reasoning — agents follow proven paths for complex workflows, not open-ended chain-of-thought
⚡
Complete-Before-Respond Mandate
No Partial Answers
All research in one turn — agents must execute every research sub-task in the same turn before composing any answer
No gap-filling with inference — partial results cannot trigger a response; agents must collect full evidence first
Drill-down enforced — initial search results are a starting point, not an answer; agents continue until the question is fully resolved
Prevents early commitment bias — eliminates the common LLM failure of anchoring on first retrieved result and ignoring contradictory evidence
🎯
Query Reformulation & Explicit "Not Found" Signaling
Precision Retrieval · No Silent Gaps
Query reformulation middleware — before hitting RAG, the agent's search query is rewritten for precision; bad queries produce irrelevant chunks which lead to hallucination
Structured "not found" responses — tools return an explicit structured signal when data is absent, not an empty result that an LLM may fill with inference
Escalation on gap — an explicit not-found signal triggers escalation to a broader search layer (web search, broader API query) rather than a fabricated answer
Turck product not-found path — three-state validation result (found / replaced / not found) forces the agent into the correct handling branch for each case
